I. How to configure/set up the project?

1. What tools you need? - top

The product is created within Android Studio IDE. You can download it from here: Download Android Studio.. In addition, on this page you can find useful information how the IDE is working, features, system requirements, tutorials and so on.

2. Import the project into Android Studio - top

Step 1: Extract the content from the .zip file that you've just downloaded from CodeCanyon. Then start Android Studio and when you see the Welcome to Android Studio screen click on Open an existing Android Studio project

Step 2: After that you will get a pop-up and you should locate where you extracted the .zip file. Select the root directory:

Step 3: When you click the OK button, Android Studio will display its home screen. There, you have to synch the project with Gradle. To do so, click on Tools -> Android -> Sync Project with Gradle Files. Of course, if you don't have Gradle on your machine, you have to install it from here.

Step 4: After Step 3, you have to see on the left side Project explorer the project's files (use the Android view) -- catalogApp item is the main project, while the other two are libraries.

Step 5: We can perform quick test, if everything is ok by selecting the catalogApp item (folder) on the left and clicking the green Run button (top navigation toolbar). This way we will launch the app on a Emulator or real device, if you have connected it to your machine. Currently we don't have real device connected (as we can see from the dialog), so we will run it on a Emulator Nexus 5 API 21 x86. Plus, this will proove that the source files are successfully compiled and there are no errors within the project.

II. How to Customize "Mobile Shop with Push Notifications"?

1. Products, Categories, Subcategories, Slider Items - top

Basically all of the data (products, categories, subcategories and slider items) is stored within a simple .xml file with a specific structure. If you want to add and/or remove products, categories and/or subcategories and to change the products within the featured slider you can do it via editing that simple .xml file. Originally it is located here: catalogApp/assets/catalog.xml.

The .xml file contains 3 main sections, as follows:

a) <categories> ... </categories>

- It contains all of your categories and subcategories. Each parent category is represented by
<category id="100"> <title> CATEGORY NAME </title> </category>
Please note that the ID must be unique amomng all other categories (subcategories)!

Each parent category can contain subcategories and you can create them as follows:
<category id="101" parentId="100"> <title> SUBCATEGORY NAME </title> </category>
Please note that the ID must be unique amomng all other categories (subcategories) and you must add second attribute parentId="100", which makes the current item, a subcategory of the matched by ID parent category. In our case it will be SUBCATEGORY fo CATEGORY with ID 100!

Check the picture below for better understanding! There we have created TWO parent categories (1st: Notebooks, 2nd: Smartphones & Tablets) and we've attached FOUR subcategories to Notebooks (e.g. Acer, Asus, Lenovo and Toshiba). Plus, we've created TWO subcategories of Smartphones & Tablets (e.g. Smartphones, Tablets. Simply said, that's the way you have to organize your categories.

Please note that you can create only one level of hierarchy (e.g. CATEGORY - SUBCATEGORY)!

b) <slider> ... </slider>

- It contains all of the products that you want to be part of the FEATURED SLIDER at the top of the home screen. Each product is represented by <product> ... </product> and contains 6 sub elements:
  • <date> - publish date;
  • <title> - the name of the product;
  • <photos> - this element contains list of the photos that you want to have for the product. Just add <photo> path or url to the image </photo> and put the name of the image file that you want to be used. Make sure to skip the file extension (e.g. .png) if you are using the local .xml. If you are using external .xml, there the file extension must be added. Always use optimized .png files;

    • Example:

  • <description> - short description of the product;
  • <condition> - new or used;
  • <color> - mention the color of the product.
All of those sub elements are mandatory and they must be closed with the related closing tag, e.g. </date>, <.title>, </photo>, </description>, </condition>, </color>. You put the value between the opening and the closing tags.

As you can see from the picture below each product has THREE required attributes:
  • id="1" - must be unique among all items;
  • category="202" - connect the product to a specific category using the category ID;
  • price="1000" - the price of the product.

- Products that are not part of the FEATURED SLIDER have to be placed within the <products> </products> tags. Each product is represented by <product> ... </product> tags and it contains 6 sub elements. Check point b) (above) to see how to build a product.

2. Settings - top

Generaly most of the settings are located within AppSettings.java class (file). All of the settings are explained in details with comments and you can easily get the idea.

Here are all of them:
  • public static final APP_THEMES CURRENT_THEME = ThemeGreen; - with this setting you can control the theme of the app. You can choose between 7 built-in themes: ThemeGreen, ThemeBlue, ThemeOrange, ThemePurple, ThemeDarkBlue, ThemeNeutral, ThemePink. Just replace the value (e.g. ThemeGreen) with the desired theme value and you are done;
  • public static final String CURRENCY = "$"; - the currency sign or code;
  • public static final double VAT = 0.2; - the VAT percentage;
  • public static final boolean useCommaForPrices = false; - control the price format. If set to false the prices will be displayed with dot format (e.g. 1000.0 ). If set to true, the prices will be displayed with comma format (e.g. 1000,00. Please note that within the XML file you MUST write the prices with dot format and they will be transformed into comma format, if this constant is set to true;
  • public static final String XMLResourcePath = "catalog.xml"; - location of the .xml file, where all of the data is stored. By default it uses the local .xml file within the assetsfolder. Of course you can use externl (web hosted) .xml file and here you can set up the path to it (e.g. http://dmb-team.com/apps/catalog/catalog_web.xml);
  • public static final String CATALOG_NAME = "Mobile Shop"; - name used within the Left Sliding Menu;
  • public static final String MAIL = "support@dmb-team.com"; - email address;
  • public static final String PHONE = "555-555-33"; - phone number;
  • public static final String SKYPE = "dmb.catalog"; - skype account;
  • public static final String FACEBOOK = "facebook.com/catalog"; - Facebook page.

3. Using External .xml file - web hosted - top

Iif you want to use external (web hosted) .xml file, just open the AppSettings.java class (file) and look for the public static final String XMLResourcePath = "catalog.xml";. Here you just have to replace the catalog.xml value with the url address of the web hosted .xml (e.g. http://mydomain.com/catalog.xml).

4. Customize the Launcher Icon (Application Icon) - top

You just need to replace the following image res/drawable-xxxhdpi/ic_launcer.png with yours. Please make sure to create an image with the same image size (width and height).
